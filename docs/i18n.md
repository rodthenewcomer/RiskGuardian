# RiskGuardian — Internationalisation (i18n)

---

## Overview

RiskGuardian is fully bilingual: **English (en)** and **French (fr)**.

Every task that adds or changes UI text **must implement both languages**. This is non-negotiable.

---

## Translation File

**Location:** `src/i18n/translations.ts`

**Structure:**

```typescript
export type Language = 'en' | 'fr'

export const translations = {
  en: { /* all English strings */ },
  fr: { /* all French strings */ }
}
```

### Namespaces

Strings are organised by feature namespace:

| Namespace | Covers |
|---|---|
| `nav` | Navigation labels (Dashboard, Risk Engine, Journal, etc.) |
| `header` | Header labels (Balance, Notifications) |
| `dashboard` | Dashboard page (KPIs, alerts, equity curve) |
| `calculator` | Risk calculator (inputs, verdict, daily limit) |
| `plan` | Trade plan HUD |
| `journal` | Journal page (trade list, calendar, filters) |
| `analytics` | Analytics page (tab labels, KPI labels) |
| `settings` | Settings page (all config labels) |
| `simulator` | What-if simulator |
| `ai` | AI Chat page (prompts, coaching labels) |
| `errors` | Error messages |
| `validation` | Form validation messages |

## How to Use Translations

### In components with Zustand `lang`

```typescript
import { useAppStore } from '@/store/appStore'
import { translations } from '@/i18n/translations'

function MyComponent() {
  const { lang } = useAppStore()
  const t = translations[lang]

  return <h1>{t.dashboard.title}</h1>
}
```

### Short inline conditionals (acceptable for 1–2 word strings)

```tsx
<span>{lang === 'fr' ? 'Gains' : 'P&L'}</span>
```

> Only use this for very short, context-specific strings. All labels, headings, hints, and CTAs go through the translations file.

## Adding New Strings

**Step 1 — Add to translations.ts under the correct namespace:**

```typescript
// src/i18n/translations.ts
export const translations = {
  en: {
    dashboard: {
      // ... existing
      weekendGapWarning: 'Open positions detected. Weekend gap risk is elevated.',
    }
  },
  fr: {
    dashboard: {
      // ... existing
      weekendGapWarning: 'Positions ouvertes détectées. Le risque de gap de weekend est élevé.',
    }
  }
}
```

**Step 2 — Use in the component:**

```tsx
<p>{t.dashboard.weekendGapWarning}</p>
```

**Step 3 — Verify FR renders** before marking done.

## French Copy Guidelines

### Pronouns

| Context | Pronoun |
|---|---|
| AI coaching messages, behavioral observations | **tu** (informal, direct) |
| Settings labels, form placeholders, formal UI | **vous** (formal) |

### Trading Vocabulary

| EN | FR (use this) | FR (avoid) |
|---|---|---|
| P&L / Profit | Gains | Profits |
| Balance | Solde | Balance |
| Risk | Risque | Risk |
| Leverage | Levier | Leverage |
| Session (trading session) | Séance | Session |
| Open trade / Position | Position ouverte | Trade ouvert |
| Lot | Lot | — |
| Win Rate | Taux de gains | Taux de victoire |
| Drawdown | Drawdown | Perte maximale (too long) |
| Stop Loss | Stop Loss | — (keep in English) |
| Take Profit | Take Profit | — (keep in English) |
| Entry | Entrée | Entry |
| Guard (daily guard) | Protection | Garde |
| Alert | Alerte | Notification |

### Tone Rules

- Coaching text: direct, imperative. "Arrête de trader le mardi." not "Tu devrais peut-être envisager..."
- Error messages: factual, no apology. "Limite journalière atteinte." not "Désolé, vous avez atteint votre limite."
- Labels: concise, no articles where English has none. "Solde" not "Le solde".
- CTA buttons: strong verb first. "Voir les analytics" not "Cliquer pour voir les analytics".

### Numbers and Currency

- Currency: `1 234,56 $` (French uses space as thousands separator, comma as decimal)
- Percentages: `62,4 %` (space before %)
- Dates: `22 mars 2026` (day month year, no comma)

## Quality Check Before Shipping

Before marking any task done that touches UI text:

- [ ] All new strings added to `translations.ts` under both `en` and `fr`
- [ ] FR copy is human-quality (not machine-translated)
- [ ] `tu` used in coaching, `vous` used in formal UI
- [ ] Trading vocabulary uses FR standard terms (see table above)
- [ ] Numbers/currency formatted for FR locale
- [ ] No hardcoded English strings in JSX
